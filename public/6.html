Guide to learning Frontend Web Development - part 2




Phase 2: Explore and Solidify

Recap: Your level at this point


- You should be comfortable with building any type of static website with html and css.

- You should know javascript pretty well and have solved at least 50 coding problems.

- You should be comfortable with dom manipulation.

- Building websites at this point should be pretty easy for you.



Things that you need to keep doing:

  • Solve 1 js coding problem each day

  • Do 2 dom manipulation exercise per week

  • Keep learning new technologies listed in Phase 2







What is phase 2?

During phase 2, you will be learning new technologies that you’ll be adding to your stack, which will help you get better as a developer and make you more productive



1. Learn Sass: (duration 1 week) : Learn

Sass - Css extension: (pre-processor) gives you more power, allows you to write modular css.

Spend 1 week on it and then go back to javascript. If ever you need it in the future, you’ll be able to pick it up easily.



2. jQuery: (duration: 1 week): build 2 projects (todolist, countdown timer)

Learn the basics. Yes JQuery is old, no one uses it, etc … . But there is a lot of codebase that still uses jQuery. What if you can find an online / freelancing job with jQuery that can make you an easy 100$ per hour. (speaking from experience). Learn the basics, know what it is, what it consists of.

Take 1 week to learn and practice, then keep using sass in all your future projects. You will improve yourself over time.



3. Learn Bootstrap: (spend 2 weeks) : build 5 projects using bootstrap

CSS framework (also has javascript). Learn to read documentation. Very good skill to have that will help you in the long run.

Note: Bootstrap will make prototyping easy for you. You can build prototypes of websites very fast, and then modify and optimize the websites for production. Keep in mind, you use bootstrap for prototyping, but not for production. It’s too heavy for production. You will convert the code into css and js for production. Also there are lots of frameworks similar to Bootstrap, but they are all similar. Once you learn bootstrap, the other are similar.



4. Learn about APIs: (duration 2 weeks) :application programming interface

A way for you to communicate to the backend, get data and display data. Build a couple of projects. (weather app, recipe app,...)

Note: Take longer if you need to, learning and knowing how to communicate with APIs will make you a better developer.

Learn about Fetch and Axios for making requests.

Once you build one or 2 projects, you will have a better understanding of APIs.



5. Gulp: (spend max 1 week): gulp is a build system that will increase you productivity

Note: you don’t need to master it, you just need to know the basics and know how it works.



6. Webpack: (spend 1 week): module bundler

Know the basics. You don’t have to master it or learn from scratch.

Most of the time, you’ll just be adding few lines of code to an existing webpack config. Learn the basics and then move on. You can learn it in depth if required at a later stage in your career. (senior dev)



7. Learn basics of backend. (1 month)

Again LEARN THE BASICS. A good frontend dev should have an idea how backend works.

Start with the Theory. Know the basics, How front end communicate with backend, sending requests, getting responses back.

Then learn FIREBASE.

It will allow you to deploy your site online for free.






Congrats. You are now a Frontend Developer. Now What ???


At this point you should be comfortable with building any type of website.

You should start looking into freelancing opportunities.

Check out my article on How to make money with what you've learnt so far



Also, you should continue learning and keep improving your set of skills. Never settle. You have to keep learning.
